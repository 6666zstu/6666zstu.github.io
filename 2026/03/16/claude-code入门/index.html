claude code入门

发表于 2026-03-16 分类于 随笔 本文字数： 1.6k 阅读时长 ≈ 6 分钟

根据 Ado Kukic 于 2026 年 1 月 1 日发布的文章 Advent of Claude: 31 Days of Claude Code 精选、汉化与重组。本文不是逐段直译，而是从 31 个分节里挑出最值得初学者先掌握的部分。

核心结论

原文最有价值的地方，不是教你几个“神奇提示词”，而是说明：Claude Code 真正强在工作流，而不只是回答能力。

新手最容易踩的坑，是把它当成一个会写代码的聊天框，随口丢一句“帮我改 bug”。这样当然也能用，但经常会出现两种情况：

  • 它不知道你的项目约定，改得不贴合
  • 它一上来就动手，方向还没对齐

所以，初学者的最佳心法只有一句：

先让 Claude 理解项目，再让它改代码；先让它出计划，再让它执行。

1. 新项目先做 /init

这是全文里我最建议立刻照做的一条。

/init 会扫描代码库，并生成 CLAUDE.md，把项目里最重要的信息整理出来，例如：

  • 怎么构建、测试、运行
  • 关键目录是做什么的
  • 代码风格和开发约定
  • 需要优先知道的架构背景

这一步的意义非常大。很多时候 Claude 写偏，不是模型差，而是它根本没被“入职培训”。
有了 CLAUDE.md，它更像一个了解团队规则的协作者，而不是纯靠猜的外援。

如果你愿意再进一步，还可以把专题规则放进 .claude/rules/，例如测试规范、安全要求、接口约束等。
但对初学者来说，先记住一句话就够了：每个项目最好都先跑一次 /init。

3. 先计划，后执行

如果只能从全文里选一个最值得养成的习惯，我会选 Plan Mode。

它的价值在于：Claude 可以先读代码、查依赖、理解结构、列出方案，但不会立刻改文件。这对新手特别友好，因为你最怕的不是 AI 不会写，而是它写得太快、改得太多。

4. 学会把会话当项目资产

原文里有几个很容易被忽视，但其实非常实用的能力：

  • --continue
  • --resume
  • /rename
  • /export

它们说明了一件事：Claude Code 不是一次性聊天，而是可以延续的工作会话。

现实开发经常是这样的：上午分析一半，下午再写；今天先定位问题，明天继续重构。
如果每次都重新解释背景，你会很累，Claude 也会不断丢上下文。

所以更好的习惯是：

  • 给大任务命名
  • 中断后优先恢复旧会话
  • 重要方案或过程导出成 markdown 留档

当你这么用时，Claude Code 才真正开始像“开发搭子”。

5. 权限边界比“全自动”更重要

原文既讲了 /sandbox，也提到了跳过权限确认的危险模式。
对初学者来说，我的建议很明确：先学会设边界，不要急着开全权限。

原因很简单。你还在磨合 Claude Code 的行为方式时，最稳妥的用法应该是：

  • 先让它读代码、查状态、跑测试
  • 对删文件、装依赖、改关键配置保持谨慎
  • 先在小任务和可回退环境里熟悉它

等你真的了解它的工作习惯之后，再逐步放宽权限也不迟。

原文还提到 hooks。你可以把它理解成“自动检查和自动拦截机制”。
比如某些操作前先跑检查，某些输出后自动记录。新手暂时不用深挖，但要知道这代表 Claude Code 不只是聊天工具，它还能接入工程化流程。

6. 真正能拉开差距的，是可复用工作流

文章后半部分还提到了不少进阶能力，我认为初学者最值得建立概念的有三类。

Headless 模式

这说明 Claude 不只能在交互界面里用，还能接进脚本和流水线。

自定义命令

把常用 prompt 存成 markdown 后，可以做成斜杠命令。
本质上，就是把你常用的“提问套路”固化成按钮。

Skills / Subagents / Plugins

这些属于更进阶的扩展能力。你可以先把它们理解成：
Claude Code 的目标不是“会聊天的 IDE”，而是“可扩展的智能开发环境”。

给初学者的 7 条落地建议

  1. 新项目先跑 /init
  2. 提问尽量附上 @文件 或 @目录
  3. 让 Claude 先做计划，再动代码
  4. 用会话恢复能力处理长期任务
  5. 默认保守权限，不要一上来开全权限
  6. 常用 prompt 逐步沉淀成命令
  7. 把 Claude 当协作者，不要只当搜索框

结语

这篇“31 天 Claude Code”最值得初学者吸收的，不是某个单独技巧，而是一整套协作思路：

先给背景，先定边界，先做计划，再执行，再沉淀成流程。

如果你把这几个动作练熟，Claude Code 就不会只是偶尔灵光一现的 AI 工具，而会逐渐变成一个稳定、可复用、能持续帮你推进项目的工程伙伴。
